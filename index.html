Mobile scaling and Modals

09 Oct

Finished up the remaining features that I wanted for my blog. One of the features that I wanted was a way to retain my unique way of displaying the date even in mobile mode. However keeping this system in a column to the side was not good as it didn’t leave a lot of room for the content of my blog post, and also the ‘buttons’ of this system did not carry across well to mobile as there is no way to hover over the buttons and so my dynamic cursors do not work. I thought of adding text that presents as a button above and below the dates, however I really didn’t like the way that this clashed with the movement of the date of an inactive blog post (as they overlap when moving). My plan for mobile was to move the date system to be sticked at the top of the screen and obscure the content as it is scrolled underneath, functionally this worked okay, however I found that the overlaid element did not match the existing aesthetic of my blog and I found it really jarring. I also had to use some unpleasant ways of aligning the text to the top such as using negative margins, as the day and the month dates are in different wrappers, and so can’t align normally. I ended up returning to just moving the dates into the content of the blog post, and having the date be unique to each blog post. This method is boring; however, it can be implemented neatly and in line with my existing content unlike the other way I was trying.

I also gave the images in my slider a dynamic width and a set height, this is so that no matter the proportions of and image, it will always align perfectly with the ones next to it, prior to this if the image was taller then it would jut out awkwardly from the stack. I also went through and applied all of the appropriate scaling to all of my elements when in mobile mode, doing this helped me change the way I was calculating the width of each of my elements to a better way, as before I was using a lot of width: 100%, and this was messing with the scaling a lot, especially for the image slider. I also added a onclick modal for my images, as while my slider is a nice way of storing multiple images in a blog post, it is not the best for viewing images. Adding the onclick modal presented three new issues however in my JavaScript.

The first was getting the modal to overlay only the image that was clicked, and to do this in a dynamic manner so that I don’t need unique ids for every one of my images as I have a lot. I fixed this by ripping code from codepen, where it uses the ‘this’ function to determine which image to display.

The second was preventing scrolling whilst the modal was opened. When I first implemented the modal I wasn’t running into the problem as I was using main as an overflow container for my blogpost, however when I refactoring my scaling I removed the overflow and instead just left the scrolling default, and this allowed for scrolling even with the modal open. I couldn’t fix this using pointer-events: none as I needed to register a click to close the modal. To fix this I went into my function that opens and closes the modal and added the overflow: hidden styling to body when the modal is opened, and then overflow: auto when it is closed.

The third problem was that because my slider already interacting with onclick for the dragging function, the image couldn’t differentiate between a click, and a click and drag and so would incorrectly display the modal when the user is trying to drag. I managed to write the code for this as the solutions I found online seemed to be overly convoluted. The code I wrote essentially records the x and y of the mouse when it is pressed down, and when it is released. If the initial x and y is equal to the end x and y, then that means that the user clicked, as the position of the mouse did not change whilst the cursor was held down. If it did change then that means that user dragged. I then used these in if statements before opening the modal.

Aside from implementing these things I also fixed a lot of small things and overall did a lot of tidying up of my code. I placed all my media query’s CSS into one big media query at the bottom and took out a lot of the placeholder divs and wrappers I had in my html.

Lastly, after resizing the viewport so much when working on the mobile version of my blog, I found that because I had a lot of subtle animations playing, I really disliked how they would be triggered on the window changing. I found some JavaScript online that disabled this, and I prefer this a lot.

Inline with this kind of fix I also added some CSS which disabled the awkward blue highlight that appeared in the mobile version, in place of this I reformatted my landing page so that size of the buttons regardless of where the text is always equal to the width of the landing page, this way I can add a onclick effect for mobile where the ‘background’ of the button flashes a slightly darker grey indicating the ‘hitbox’ of the button. Initially I was planning on changing this back to how I had it originally on the desktop version, where you had to hover your cursor of the text to get the hover effect, however I ended up really liking seeing the hover effects just when the cursor was at the same height of the button. This also let me implement my custom cursor to the blog button which is an arrow aiming down. I had thought of doing this earlier, but I figured that the overlapping lines of the hover effect of the button combined with the custom cursor would be too much.

Blog navigation and the right time to start coding

02 Oct

Today I figured that I should probably spend some time working on our group project as we are supposed to have started coding it by now. I went through and built out a very rough framework. I used the same technique I used in my last iteration of this in order to get the pages on one big ‘screen’ that can be scrolled left and right, which is using and overflow wrapper, and aligning all of the different pages in grid columns which a width of 100vw. This has its issues but its and easy way to get the content out there so that if the other want to start doing work they can start on any of the pages and don’t have to wait for me to create the final system with all of its functionality. After doing this I pushed this to the master of our git project, and then made branches for everyone so that if they would like they can start working on the content of our website.

I spent the rest of today polishing the scrolling effect that I finally got working yesterday and bringing across from my test website to my blog. The first thing I did was apply the same effect to the months as well as the day so that they can scroll individually. I’m sure there is a way to optimise the JavaScript for this, but essentially all I did was copy the same code, and use different variables, in this case it was sxnday/sxnmonth, and idxday/idxmonth. I should also explain a bit of how this works, because while I started with existing code, I ended rewriting the whole thing anyway, however using that existing code helped with me understanding the logical process of what needs to happen. Essentially an array is initialised of all the sections, day or month. A new variable is then made = 0, idx, then by manipulating this idx variable and then using it as the value for the array like sxn[idx] we can ‘select’ a section. There are two ways to change the value of idx, the first is by using the next and back buttons which respectively add 1 or subtract one, idx++/idx--, the second is by scrolling, for this I created an event listener, that when the user scrolls, it records the distance from the top of the current ‘active’ section and the top of the viewport. We can then use if statements to say that if the bottom of the active section is negative, and therefore greater than the top of the viewport, the active section must have been scrolled past entirely, in which case we can add 1 to idx, and set the section below the one just scrolled to active. This then repeats and is also mirrored for when the top of the active section is below the bottom of the viewport, we can subtract 1 from idx.

Once we have established which section is ‘active’ we can then run a function that when one of the next or back buttons is clicked, it will scroll to either idx++ or idx--. I’m really proud to have made this system as it turned out be quite complex, and this is my first time doing pure JavaScript.

To polish this effect I emulated an effect from one of my favourite websites https://dasprogramm.co.uk/, where they use a carousel with buttons that change the cursor to an arrow in the direction of the movement of the carousel. I placed this new style of buttons on top of the columns for the dates at all times, and then used grids to place the next buttons on the bottom half of the screen and the back buttons on the top half of the screen. This resulted in a fantastic and in my view very intuitive system of navigation.

Blog success

01 Oct

This morning I finally got this effect that I have been working on for my blog complete. I knew that with my new date system I wanted a way to quickly scroll through the dates of blogs as opposed to scrolling through every single blog post. My solution to this was making forwards and back buttons that scroll and entire month, I implemented this very simply at the beginning however the big problem I was running into was that if you clicked the next button and got taken to blog post 2 for example, and then manually scrolled down to blog post 5. The buttons wouldn’t recognise that change and therefore if you then clicked next you would be taken to back up to blog post 3, as that is 1 greater than the blog post that the button last took you to. This problem meant that I needed to create a system where the blog post that is currently in view no matter how you got to it, would be the post that would be used to reference where forwards and back should be from.

To avoid having to deal with unnecessary issues I made new test website to create this effect. In the current iteration of it I have it set up so that as you scroll down, the section that is currently intersected by the top of the viewport is assigned as the active section indicated by its red stroke. The buttons at the top then set the active section to either +1 or -1 from the current active, and then smooth scroll to it.

I am starting this way too late

10 Aug

I’m doing most of the ‘designing’ of the wireframes tonight. My first and in my opinion best of the two wireframe ideas is a sort of backtracking system. Where the user starts at their last year for their chosen degree, and then as they select papers for that year, the pre requisite papers that they would have needed to have taken in order to take those papers in their last year are automatically added to their previous years and a line is drawn between each of the classes.

The intentions behind this design is that by the end of selecting their papers the user will have a ‘tree’ like diagram that represents the direction that they have taken each course.

My other idea a lot simpler and more traditional, there is a navbar at the top which allows the user to switch between years and the select degree page. On each page for a specific year the user can drag and drop papers from a search box and into tabs for each of the trimester. As they do so their point totals and requirements will change dynamically on a tab on the same page. The intention behind both of my designs is to give a sense of tactility and weight to their selections outside of dropdown menus, and to very clearly show the prerequisite papers that need to be taken in order to complete their selected degree.

Blog restyle

26 Jul

After working on the structure of my blog yesterday I started looking at the styling of the whole page, as well as the structure of each blog post. With my current layout of a lot of nested divs each with their own padding I was wasting a lot of space. I decided to remove the colour from each of the different div, so that a smaller margin was less jarring, I then gave each element its own div as opposed to nesting all of the 'content' together. This way I can freely reorder how the information is stacked. I also changed from using the grid system to center and scale the blog posts proportionally, as when using fractions, the blogs would become too thin when the browser was rescaled. I changed to instead using a min and max width, and then a margin on the sides of the scrollbar div. This is so that the blogs will stay at a constant size when being scaled, until the margin pushes them closer together.

Style wise I changed from my boxy appearance as that was really just there to see the margins for each div. I'm a fan of minimalist websites so I went for an incredibly boring theme of black and white, using the font Helvetica. I restyled the scrollbar to match this aesthetic and added a single line border on each blog post to easily differentiate between blog posts.

Blog restructure

25 Jul

Worked a lot more on the structure of blog itself, I had a rough idea of what I wanted to do, such as the best way to implement the stickied header at the top. I disliked the regular way of achieving this of just giving the header a stickied position because I wanted the header in both the home page and reflections page to be inline, however with more posts the reflections page had a scrollbar and the home didn't, causing them to be out of line.

The way I ended up getting around this was actually setting the wrapper div to be the exact viewport size, so that a scrollbar is not generated. And then wrapping the blog posts within another div, this time with overflow, this meant that the header stayed constant and I could still scroll through the blogs nicely, this also meant that the scrollbar was now nicely offset from the side of the page, and I can place it neatly in-line with the header with a consistent margin.
